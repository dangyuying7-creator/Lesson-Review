# 销转直播课数据诊断工具

## 项目概述

本地离线前端 Web 应用，供**销售转化运营**人员使用，对直播课堂进行数据量化诊断。
核心功能：上传课堂评论数据 + 出勤转化数据 → 自动评分 → 生成排查报告。

**技术栈**：纯 Vanilla JS，无构建工具，直接用浏览器打开 `index.html` 即可运行。
- Tailwind CSS（CDN）
- Chart.js 4.4（CDN）
- SheetJS xlsx-0.18.5（jsDelivr CDN，备用：本地 `js/xlsx.full.min.js`）

## 核心词汇定义

### 出勤与转化

| 术语 | 定义 |
|---|---|
| 直播出勤5分钟 | 直播出勤时长 ≥ 5分钟的用户数 |
| 直播出勤10分钟 | 直播出勤时长 ≥ 10分钟的用户数 |
| 直播出勤30分钟 | 直播出勤时长 ≥ 30分钟的用户数 |
| 5分钟转化率 | 出勤 ≥ 5分钟 且 当天转化的用户数 ÷ 出勤 ≥ 5分钟的用户数 |
| 10分钟转化率 | 出勤 ≥ 10分钟 且 当天转化的用户数 ÷ 出勤 ≥ 10分钟的用户数 |
| 30分钟转化率 | 出勤 ≥ 30分钟 且 当天转化的用户数 ÷ 出勤 ≥ 30分钟的用户数 |
| 转化 | 低价课结束后当天下单正价课（仅计当天，不含跨日） |
| 5分钟留存率 | 出勤 ≥ 10分钟人数 ÷ 出勤 ≥ 5分钟人数，衡量开场吸引力 |
| 30分钟留存率 | 出勤 ≥ 30分钟人数 ÷ 出勤 ≥ 10分钟人数，衡量课程中段粘性 |
| 同主讲转化率 | 当节课转化订单中，选择本节课主讲老师正价课的比例 |

### 评论分析

| 术语 | 定义 |
|---|---|
| 总评论数 | 评论表中的全部原始行数 |
| 有效评论数 | 过滤掉助教消息、课堂秩序类、噪音后剩余的学员评论条数 |
| 噪音评论 | 字数 < 3 / 纯数字标点 / 重复字符比例 > 65% 的评论 |
| 好评 | 命中正面关键词且未被否定前缀抵消、非疑问句的评论 |
| 差评 | 命中负面关键词的评论 |
| 中性评论 | 既无好评词也无差评词，或为疑问句的有效评论 |
| 好差评比 | 好评数 ÷（好评数 + 差评数），仅统计有明确情感倾向的评论 |
| 人头数 | 发过差评的不重复用户数（按昵称去重） |
| 助教消息 | 单场发言 ≥ 15 条的高频用户 / 引用回复格式 / 命中助教话术关键词 |
| 课堂秩序类 | 学员之间的劝退/互怼评论，不反映对老师教学的评价 |

## 评分结构（总分 100 分）

### 无转化反馈时
| 模块 | 满分 | 说明 |
|---|---|---|
| 留存转化分 | 60 | 5min留存率20 + 30min留存率25 + 5min转化率5 + 30min转化率10 |
| 互动评论分 | 30 | 有效评论数10 + 好差评比20 |
| 同主讲转化率 | 10 | 不归一化，直接累加 |
| **合计** | **100** | |

### 有转化反馈时（归一化）
基数 = 60 + 30 + 10 + 20 = 120，总分 = 原始得分 × 100 / 120

## 评分规则细节

### 转化率评分（5档，分学科动态标准）
在 `config.js` → `SUBJECT_CONV_TIERS` 配置，阈值依据近一年历史数据分位数（P25/P50/P75）：

| 区间 | 得分比例 |
|---|---|
| ≥ P75（满分线） | 100% |
| ≥ P50（良好线） | 80% |
| ≥ P25（合格线） | 60% |
| > 0，低于合格线 | 线性折算，最高 40%（有转化即有分） |
| = 0 | 0 分 |

> 最后更新：2026-06-09，原始数据来源：主讲当堂课_24小时出勤转化率（111条）

### 留存率评分
- 5分钟留存率（满20分）：≥90% 满分，否则按比例折算
- 30分钟留存率（满25分）：≥60% 满分，否则按比例折算

### 有效评论数得分（满10分）
- 满分线 = 5分钟出勤人数 × 1.0（1人1条即满分）
- 按实际有效评论数与满分线的比例折算

### 好差评比得分（满20分）
- 负面评论 ≥ 正面评论 → **直接 0 分**
- 正面 > 负面且好差评比 ≥ 90% → 20 分
- 否则 → 20 × 好差评比%

### 同主讲转化率（满10分）
| 同主讲占比 | 得分 |
|---|---|
| ≥ 90% | 10 |
| ≥ 75% | 8 |
| ≥ 60% | 5 |
| ≥ 40% | 2 |
| < 40% | 0 |

### 转化反馈（满20分，可选）
五档评分（5→1分），加权平均 avg，得分 = (avg-1)/4 × 20

## 评论分析逻辑（analyzer.js）

### 过滤顺序（以下均不计入学员评论统计）
1. **助教/场控**：发言 ≥15条的高频用户 / 引用回复格式 / 话术关键词
2. **课堂秩序类**：针对其他学员的劝退/互怼（见 `ORDER_PATTERNS`）
3. **噪音**：字数<3 / 纯数字标点 / 重复字符>65%

### 分类规则
- **疑问句**（末尾含吗/呢/？）→ 中性
- **否定前缀**（命中位置前2字含不/没/别/未）→ 正面词失效
- **宽泛正面词**（棒/赞/好等）→ 需同时含课程锚词（老师/讲/课/题等）才生效
- **负面优先**：正负词数量相同时判为差评

### 时段分析数据一致性
`analyzeByTimeSlots()` 使用与主循环**完全相同**的过滤条件（重建 assistantIds + isOrderComment + isNoise），确保时段差评数 = 差评原文条数。

## 报告生成逻辑（report.js）

三个部分：
- **讲课优点**：留存率/转化率/评论达标时输出正向描述 + 好评关键词 + 好评原文
- **不足之处**：未达标指标 + 时段差评集中分析 + 差评关键词 + 差评原文（按人头合并）
- **请老师自查**：结构化卡片，不给教学建议，只输出排查问题清单

### 请老师自查卡片结构
每张卡片包含：图标 + 标题 + 指标说明 + 得分（可选） + 排查问题列表

卡片类型及触发条件：
| 卡片 | 触发条件 | 边框颜色 |
|---|---|---|
| 学科·30min转化率 | 低于良好线（P50） | 橙/红 |
| 留存预警·5→10分钟 | 5分钟留存率不达标 | 橙 |
| 留存预警·10→30分钟 | 30分钟留存率不达标 | 橙 |
| 评论区排查 | 命中差评关键词 / 口碑预警 | 橙/红 |

评论区排查卡内部按信号类型分节：语速过快 / 学员听不懂 / 例题偏难 / 课堂无聊 / 音视频问题 / 口碑预警（差评≥好评）

> 注意：运营身份无法给专业教学建议，"请老师自查"栏目的设计是传递信号，让老师自行对照录像判断。

## 常见维护场景

### 增加新的助教话术关键词
编辑 `config.js` → `ASSISTANT.contentPatterns`，添加正则表达式。

### 更新学科转化率阈值
编辑 `config.js` → `SUBJECT_CONV_TIERS`，格式为 `[满分线%, 良好线%, 合格线%]`。

### 新增学科
在 `SUBJECT_CONV_TIERS` 中添加对应学科的 `conv5` 和 `conv30` 阈值；
在出勤数据 Excel 的「学科」列填写与配置完全一致的名称即可自动匹配。

### 调整评分权重
- 留存转化各项上限：直接改 `scorer.js` 中对应的 `maxXxx` 变量
- 评级分数线：改 `config.js` → `RATINGS`

## 数据流说明

```
上传 Excel/JSON
    ↓
app.js parseAttendance / parseCommentFile
    ↓ State.attendanceData（含 subject）/ State.commentRows
generateReport()
    ↓ readManualAttendance()（从 State.attendanceData 补入 subject）
Analyzer.analyzeComments(rows)
    ↓ analysis（effective/positive/negative/timeSlots...）
Scorer.calcScore(attendance, analysis, feedbackCounts, sameTeacher)
    ↓ scoreData（retention.subject → 匹配 SUBJECT_CONV_TIERS）
Report.generateFullReport(teacherInfo, attendance, analysis, scoreData)
    ↓ { text, strengths, weaknesses, suggestions, suggestionGroups }
updateScorePanel / updateAnalysisContent / updateReportOutput
```

## 注意事项

- 本工具**纯本地运行**，不联网、不上传数据，适合处理含学员信息的内部数据
- 首次打开需联网加载 CDN 资源（Tailwind / Chart.js / SheetJS），之后浏览器会缓存，可离线使用
- 评论分类当前基于**关键词匹配**，后续计划接入语义理解模型以提升准确率
- 图片上传（同主讲转化率 BI 截图）仅供视觉参考，不做自动识别，需手动填入数字
- Excel 解析依赖 SheetJS；若 CDN 加载失败（国内网络问题），可将 `xlsx.full.min.js` 下载到 `js/` 目录并修改 `index.html` 引用为本地路径
